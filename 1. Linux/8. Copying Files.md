- If a copy of a file is created before changes are made, then it is **possible to revert back to the original.**
- A copy of a file can be used to **transfer a file to removable media devices**.
- A copy of an existing document can be **used as a template for a new document.**

- `cp <source> <destination>`
- I am copying the content of /etc/passed to my currant directory (`.`) 

> To use copy command ( a [[File-management-command]]), There are certain permissions you need to have 
> **Execute** permission to the directory the file you are copying from
> **Read** permission to the file you are copying 
> **Read and Write** permission to the directory the file is being copied. 

![](../Assets/Screenshot%202026-02-26%20180050.png)
- here we have read permission to the file (/etc/passwd)
- and execute permission to the directory the file belongs.  [[cheking-permission-of-directory]]

> Typically, there are two places where you should **always have write and execute** permission on the directory: your home directory and the `/tmp` directory.

#### Why need a execute permission to a directory ? (we're not executing anything...)

> First `x` does not mean executing/running a program to a directory. 
> It means **You can enter the file.** 

![](../Assets/Screenshot%202026-02-26%20181410.png)
- Here Documents directory has no execute permission initially. 
- I tried to enter into it using `cd` but permission is denied. 
- I then gave the permission using `chmod` to successfully access it. 

> Therefor to copy a file you must first traverse/enter through the directory so you must need the execute permission.
