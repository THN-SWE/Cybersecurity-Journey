#### ### Match a Single Character With `.`

> One of the most useful expressions is the period `.` character. It will match any character except for the new line character. The pattern `r..f` would find any line that contained the letter `r` followed by exactly two characters (**which can be any character except a newline**) and then the letter `f`:

![](../Assets/Screenshot%202026-02-28%20142053.png)

#### > Match a Single Character With `[]`

> The square brackets `[ ]` **match a single character** from the list or range of possible characters contained within the brackets.

![](../Assets/Screenshot%202026-02-28%20152705.png)
- This prints all the lines with at least one digit in it.

>  **What if i want to print the lines without any numbers in it ?** 
![](../Assets/Screenshot%202026-02-28%20152906.png)
- This is not expected. 
- `^` → start of line
- `[0-9]` → one digit
- So this pattern means: “Match lines that start with a digit.”
- **very intuitive**

![](../Assets/Screenshot%202026-02-28%20154137.png)
- well then this should work right  ? NO. 
- Inside square brackets `[ ]`, the `^` means **NOT**.
- `^[0-9]` means : match any single character that is NOT a digit.  ie: Print lines that contain at least one character that is not a digit.
