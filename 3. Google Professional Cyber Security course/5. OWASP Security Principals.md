> The **OWASP** security principles are **guidelines for building secure applications**

- It stands for *Open Worldwide Application Security Project*
- OWASP is not just about "top 10 web vulnerabilities" It is also about **Principals that guide secure system design.** 
- Following are the main ones. Checkout official site to learn more: [OWASP-Principals](https://devguide.owasp.org/en/02-foundations/03-security-principles/)

1️⃣ **Least Privilege**  
Give users and processes only the access they need.  
*Example:* Regular users read files, admins edit system configs.

2️⃣ **Defense in Depth**  
Use multiple layers of security to prevent single points of failure.  
*Example:* Firewall + passwords + MFA + monitoring.

3️⃣ **Fail Securely**  
Systems should fail in a secure way.  
*Example:* If login server crashes → deny access.

4️⃣ **Don’t Trust User Input**  
Always validate and sanitize inputs from users.  
*Example:* Prevent SQL injection in forms.

5️⃣ **Keep Security Simple**  
Avoid unnecessary complexity.  
*Example:* Use a simple, well-tested authentication system.

6️⃣ **Separation of Duties**  
Critical tasks should require multiple people or approvals.  
*Example:* One employee creates payment, another approves.

7️⃣ **Secure by Default**  
Systems should be secure out-of-the-box.  
*Example:* Closed ports, strong passwords, logging enabled.

8️⃣ **Keep Security Visible**  
Make security actions and logs clear for monitoring.  
*Example:* Logs of failed logins, admin actions, suspicious activity.

> **Memory Trick:** “Least defense fails, simple defaults separate duties.”
> 
- Least → Least Privilege
- Defense → Defense in Depth
- Fails → Fail Securely
- Simple → Keep Security Simple
- Defaults → Secure by Default
- Separate Duties → Separation of Duties
