> Security Information and Event management 

- A SIEM tool is used by Soc analysts to monitor and detect cyber threats in real time. 
- SIEM collects log information from all kind of systems and analyzes then alert the team if anything suspecious happens. It collects logs from, 
	- [Intrusion Detection System](../../Misc/Intrusion%20Detection%20System.md)
	- [Intrusion Prevention System](../../Misc/Intrusion%20Prevention%20System.md)
	- [Firewall](../../Misc/Firewall.md)
	- DNS logs 

> A **SIEM centralizes logs from many systems and automatically correlates them**, allowing security teams to quickly detect attacks, troubleshoot issues, and understand the full picture instead of checking logs manually.

#### 3 Types of SIEM tools 

##### Enterprise SIEM

> These are SIEM software built by enterprises to host on your own server.

- Splunk Enterprise Security
- IBM QRadar
- ArcSight ESM
##### Cloud SIEM

> There are 2 types, Cloud hosted and Cloud native
> Cloud hosted means you run your tool on cloud server than your own
> Cloud native means the SIEM is specifically built for the cloud.

- Microsoft Sentinel (Cloud native)
- Google Chronicle (Cloud native)
##### Open Source SIEM

- Wazuh
- Elastic Security
#### MAIN TYPES OF LOG DATA IN SIEM

| Log Type             | What It Records                           | Used To Detect                       |
| -------------------- | ----------------------------------------- | ------------------------------------ |
| Authentication Logs  | Logins, logouts, failed attempts          | Brute force, credential abuse        |
| Network Logs         | Source IP, destination IP, ports, traffic | Suspicious connections, scanning     |
| Endpoint Logs        | Processes, commands, file activity        | Malware, attacker tools              |
| Application Logs     | User activity inside applications         | Insider threats, abnormal usage      |
| Security Device Logs | Firewall, IDS/IPS, antivirus alerts       | Active attacks, malware              |
| System Logs          | OS events, services, account creation     | Privilege escalation, system changes |
| Cloud Logs           | Cloud user actions, API calls, resources  | Unauthorized cloud access            |
