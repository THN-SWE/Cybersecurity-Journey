- **OS hardening = Making an Operating System more Secure by reducing its [Attack surface](../../Misc/Attack%20surface.md)**

- Making the system to only have necessary stuff open && strengthen available entry points. 
- Fewer entry points = Less chance of exploitation and easier to find/pinpoint threats. 

#### Common OS Hardening Actions

1. Disable unnecessary services : turning off ftp, telnet if no needed. 
2. Strong authentication system 
3. Patch and Updates regularly. 
4. Least Privilege principle : Users only get permission for what they allowed to do. 
5. Firewall configuration.
