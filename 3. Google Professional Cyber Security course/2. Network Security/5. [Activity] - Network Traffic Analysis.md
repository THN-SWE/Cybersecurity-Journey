#### Scenario 
> You are a cybersecurity analyst working at a company that specializes in providing IT services for clients. Several customers of clients reported that they were not able to access the client company website www.yummyrecipesforme.com, and saw the error “destination port unreachable” after waiting for the page to load.

- So I attempted to visit the website and captured the traffic with **tcpdump** analyzer. 

#### Behind the scene 

1. My browser sends a query to DNS server via [[UDP Protocol]]. (To retrieve the IP address of the webiste).
2. Then it uses the retrieved IP address to make the HTTPs request to the site. 
3. Here, when I send the UDP packets to the DNS server I am gettingi [ICMP](../../Misc/ICMP.md) packets containing error message. 

#### Part 1: Provide a summary of the problem found in the DNS and ICMP

1.  **The UDP protocol reveals that** : port 53 is not responding. 
2. **This is based on the results of the network analysis, which show that the ICMP echo reply**
**returned the error message:**  port 53 is unreachable. 
3. **The port noted in the error message is used for:** Handling DNS requests ? 
4. **Most likely the problem is ?** : The DNS server is down.

#### Explain your analysis of the data and provide at least one cause of the incident.

1. **Time incident occurred**: 13.24
2. **Explain how the IT team became aware of the incident:** This incident was reported by several customers. 
3. **Explain the actions taken by the IT department to investigate the incident:** Analyst used conducted packet sniffing tests using tcpdump to see the logs and found out the that DNS server is not correctly responding back. 
4. **Note key findings of the IT department's investigation :** There is an issue with the DNS server. 
5. **Note a likely cause of the incident:** The DNS server must have led down by any attack like DoS or the DNS firewall blocking the requests for some reason. 

#### What are the Next steps now that we found out the cause is DNS server. 

- Next step is troubleshooting : to see if the DNS server is down OR the traffic to port 53 is blocked by [Firewall](../../Misc/Firewall.md) by misconfiguration.  
- If the DNS server is down i could be a DoS attack.
