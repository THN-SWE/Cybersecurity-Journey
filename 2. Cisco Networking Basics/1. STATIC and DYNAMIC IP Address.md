 - There are 2 ways one IP address is assigned in a network
 - Static and Dynamic

#### STATIC IP

> With a static assignment, the network administrator must manually configure the network information for a host.

- **IP address** - This identifies the host on the network.
- **Subnet mask** - This is used to identify the network on which the host is connected.
- **Default gateway** - This identifies the networking device that the host uses to access the internet or another remote network.

- There are some advantages and disadvantages in using Static IP addressing. 
- For devices like printers, servers, CCTV, or other networking devices that need to be accessible to clients on the network, Having dynamic IP isn't idea. 
- Easy Remote Access - For services like VPN , Remote desktop, Static Ip is suitable, since you don't have to check the New IP. 
- In other hand Static IP is not that scalable - In a big network you can't manually assign IP to every device, very inefficient and Time consuming. 

#### DYNAMIC IP 

- In a local network most times new devices join, some disconnects.. user population always changes right. 
- So It would be efficient to assign IPs to them automatically
- This is done using [[DHCP-Protocol]] AND DHCP server. 

#### DHCP Server (Dynamic Host Configuration Protocol)

- The DHCP server is reponsible for assigning IPv4 address to devices in the network. 

![](../Screenshot%202026-03-04%20183130.png)
- As you can see every time of network has a DHCP server. 
- In a home local network the Wifi-router is typically going to act as the dhcp server. 
- In mobile network the dhcp server is in the ISP side.
